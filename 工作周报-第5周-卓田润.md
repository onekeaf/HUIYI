## 一、实习内容和收获

本周完成了后端剩余模块的开发，并开始前端编码实现：

**1. 药品信息模型创建及查询接口实现（3小时）**
- 创建 DrugModel 数据模型，包含药品与销售地点的多对多关联关系
- 实现 DrugMapper 接口和 XML 映射文件，使用嵌套查询实现关联数据查询
- 实现 DrugService 分页查询和 DrugController 查询接口

**2. 药品信息添加、修改、删除及图片上传接口实现（6小时）**
- 实现药品信息的增删改接口
- 实现药品图片上传功能，使用 MultipartFile 接收文件并保存到本地目录
- 配置 WebConfig 静态资源映射，实现图片的 HTTP 访问
- 处理药品与销售地点的多对多关联关系

**3. 必备材料模型创建及查询接口实现（6小时）**
- 创建 MaterialModel 数据模型和 MaterialParam 参数类
- 实现 MaterialMapper 接口和 XML 映射文件
- 实现 MaterialService 分页查询和 MaterialController 查询接口

**4. 必备材料添加、修改、删除接口实现（6小时）**
- 实现必备材料的增删改接口
- 完成接口测试验证

**5. 用户菜单管理实体类创建及菜单列表接口实现（3小时）**
- 创建 PermissionModel 和 MetaModel 数据模型
- 实现 PermissionMapper 接口，根据用户角色查询对应的菜单权限
- 实现 PermissionController 菜单列表获取接口

**6. 第二阶段评审（2小时）**
- 参加第二阶段评审会议，展示后端接口开发成果
- 使用 Swagger 文档和 Postman 演示各接口功能
- 接收评审意见，准备进入前端开发阶段

**7. 登录页面设计和前端登录功能实现（6小时）**
- 设计登录页面 UI，使用 Element UI 组件实现表单布局
- 实现前端登录逻辑，通过 Axios 调用后端登录接口
- 使用 Vuex 管理登录状态和 Token 存储
- 实现路由守卫，未登录用户自动跳转到登录页

**收获：** 本周完成了后端所有模块的接口开发并通过评审，同时开始了前端开发。通过实现文件上传功能，学习了 Spring Boot 处理文件上传的方式。前端方面，深入理解了 Vuex 状态管理和 Vue Router 路由守卫的使用。

## 二、劳动任务和体会

本周是承上启下的一周，既完成了后端的收尾工作，又开始了前端的开发。从后端切换到前端开发，需要转换思维方式——后端关注数据处理和业务逻辑，前端关注用户交互和界面展示。这种全栈开发的体验让我对软件系统有了更全面的认识。
