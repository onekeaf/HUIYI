## 1 项目名称及描述

### 1.1 项目名称

医药信息管理系统

### 1.2 项目描述

医药信息管理系统是一套面向医药行业的综合管理平台，旨在实现医药公司、药品、销售地点、医生、医保政策、必备材料等信息的统一管理与展示。系统采用前后端分离架构，后端基于 Spring Boot + MyBatis + MySQL 技术栈提供 RESTful API 接口，前端基于 Vue.js + Element UI + Vuex 技术栈实现单页面应用（SPA）。

系统支持两种用户角色（管理员和医生），通过 Token 认证和角色权限控制实现不同角色的功能差异化。管理员拥有所有模块的增删改查权限，医生角色仅拥有查询权限。系统包含数据面板、医药公司管理、销售地点管理、城市信息管理、药品信息管理、医保政策管理、医药公司政策管理、医生信息管理、必备材料管理等核心功能模块，并扩展了 ECharts 数据可视化和高德地图集成功能。

**6. 药品信息管理**
- 药品信息的查询、新增、修改、删除
- 药品图片上传与展示

**7. 医保政策管理**
- 医保政策信息的查询、新增、修改、删除

**8. 医药公司政策管理**
- 医药公司政策信息的查询、新增、修改、删除

### 2.4 非功能性需求

#### 2.4.1 性能需求

- 系统响应时间：普通查询接口响应时间不超过 500ms
- 分页查询：支持大数据量的分页查询，每页默认 5 条数据
- 并发处理：支持多用户同时在线操作
- 前端页面加载：首屏加载时间不超过 3 秒

#### 2.4.2 安全性需求

- 用户密码加密存储
- 基于 Token 的身份认证机制，Token 失效后自动跳转登录页
- 基于角色的访问控制（RBAC），不同角色访问不同的接口和页面
- 接口层面使用 @RolesAllowed 注解进行权限校验
- 前端通过路由守卫防止未登录用户访问系统页面
- 跨域请求配置（CORS），限制合法的请求来源

## 3 设计思想

### 3.1 系统架构设计

系统采用**前后端分离**的 B/S 架构，整体架构分为三层：

```
┌─────────────────────────────────────────────┐
│                  前端层（Vue.js）              │
│  Vue.js + Element UI + Vuex + Vue Router     │
│  ECharts + 高德地图 JS API                    │
├─────────────────────────────────────────────┤
│                  后端层（Spring Boot）         │
│  Controller → Service → Mapper               │
│  Spring MVC + MyBatis + PageHelper            │
│  Token认证 + 角色权限控制                      │
├─────────────────────────────────────────────┤
│                  数据层（MySQL）               │
│  MySQL 8.0 数据库                             │
└─────────────────────────────────────────────┘
```

- **前端层**：使用 Vue.js 构建单页面应用，Element UI 提供 UI 组件，Vuex 管理全局状态，Vue Router 管理路由，Axios 发送 HTTP 请求
- **后端层**：使用 Spring Boot 构建 RESTful API，采用 Controller-Service-Mapper 三层架构，MyBatis 进行数据库操作，PageHelper 实现分页
- **数据层**：使用 MySQL 8.0 存储业务数据

### 4.1 用户登录与权限管理模块

**后端关键代码：**

登录接口通过用户名和密码验证用户身份，验证成功后生成 Token 并返回用户信息和菜单权限。

```java
// AccountController - 登录接口
@PostMapping("/login")
public Msg login(@RequestBody AccountParam param) {
    return accountService.login(param);
}
```

权限控制通过 `@RolesAllowed` 注解实现，限制不同角色访问不同接口：

```java
@RolesAllowed({"1", "2"})  // 管理员和医生都可访问
@GetMapping(value = "")
public Msg getDashboardController() {
    ObjectNode node = dashboardService.getDashboardData();
    return Msg.success().data("data", node);
}
```

**前端关键代码：**

前端通过 Vuex 管理登录状态，登录成功后将 Token 存储到 localStorage，并通过路由守卫控制页面访问：

```javascript
// router/index.js - 路由守卫
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem("token");
  if (to.path == "/user/login" && token) {
    next("/");
  } else if (to.path !== "/user/login" && !token) {
    next("/user/login");
  } else {
    next();
  }
});
```

动态路由通过 `routeParse.js` 将后端返回的菜单数据转换为 Vue Router 路由结构：

```javascript
// utils/routeParse.js
function tree(data, arr) {
  data.forEach((datas, index) => {
    arr.push({
      path: datas.path,
      name: datas.name,
      component: datas.component == "Layout"
        ? Layout
        : () => import(`../views/${datas.component}/index.vue`),
      meta: { title: datas.meta.title },
      children: [],
    });
    if (datas.children) {
      arr[index].children = tree(datas.children, []);
    }
  });
  return arr;
}
```

```javascript
        zoom: 11,
        center: [104.06707, 30.660842],
      });
      this.map.on("click", (e) => {
        if (this.addStatus == 1) {
          this.creatLocation(e.lnglat.getLng(), e.lnglat.getLat());
        }
      });
    });
}
```

**页面效果：** 支持列表/地图视图切换，列表视图展示药店信息表格，地图视图展示所有销售地点标记，点击地图可新增地点，点击标记可修改地点信息。

### 5.3 项目运行说明

**后端启动：**
1. 导入数据库脚本 `bin_text.sql` 和 `sales.sql` 到 MySQL
2. 修改 `application.yml` 中的数据库连接配置
3. 修改 `WebConfig.java` 中的图片存储路径
4. 使用 IDEA 运行 Spring Boot 主类，启动后端服务（端口 8080）

**前端启动：**
1. 进入 `medical-vue` 目录
2. 执行 `npm install` 安装依赖
3. 执行 `npm install echarts@4.9.0 @amap/amap-jsapi-loader --save` 安装图表和地图依赖
4. 执行 `npm run serve` 启动前端开发服务器
5. 浏览器访问 `http://localhost:9092`
