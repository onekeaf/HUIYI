## 一、实习内容和收获

本周继续后端编码实现，完成了多个业务模块的接口开发：

**1. 医药公司政策模型创建及查询接口实现（3小时）**
- 创建 CompanyPolicyModel 数据模型和 CompanyPolicyParam 参数类
- 实现 CompanyPolicyMapper 接口和 XML 映射文件
- 实现 CompanyPolicyService 分页查询和 CompanyPolicyController 查询接口

**2. 医药公司政策添加、修改、删除接口实现（6小时）**
- 实现医药公司政策的增删改接口
- 处理政策与公司的关联关系
- 完成接口测试验证

**3. 城市信息模型创建及查询接口实现（3小时）**
- 创建城市信息实体类和数据模型
- 实现 CityMapper 接口和 XML 映射文件，支持模糊查询
- 实现 CityService 和 CityController 查询接口

**4. 城市信息添加及删除接口实现（3小时）**
- 实现城市信息的添加和删除接口
- 完成接口测试

**5. 医保政策模型创建及查询接口实现（3小时）**
- 创建 MedicalPolicyModel 数据模型和相关参数类
- 实现 MedicalPolicyMapper 接口和 XML 映射文件
- 实现分页查询接口

**6. 医保政策添加、修改、删除接口实现（6小时）**
- 实现医保政策的增删改接口
- 完成业务逻辑处理和接口测试

**7. 销售地点模型创建及查询接口实现（3小时）**
- 创建 Sale 实体类和 SaleEntity
- 实现 SaleMapper 接口和 XML 映射文件
- 实现 SaleService 分页查询和 SaleController 查询接口

**8. 销售地点添加、修改、删除接口实现（6小时）**
- 实现销售地点的增删改接口
- 完成接口测试验证

**收获：** 本周完成了多个业务模块的后端接口开发，通过反复实践 CRUD 操作，对 MyBatis 的 XML 映射配置、动态 SQL 编写更加熟练。同时对 RESTful API 的设计规范有了更深的理解，能够快速高效地完成标准化的接口开发。

## 二、劳动任务和体会

本周的工作量较大，需要完成多个模块的后端接口开发。通过这些重复性但又各有差异的开发工作，逐渐形成了自己的编码习惯和开发节奏。体会到了软件开发中"熟能生巧"的道理，随着对框架和工具的熟悉，开发效率明显提升。
