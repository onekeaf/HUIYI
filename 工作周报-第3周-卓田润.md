## 一、实习内容和收获

本周进入项目开发阶段，主要完成了后端核心模块的编码实现：

**1. 用户模型创建及登录权限验证（6小时）**
- 创建用户实体类和数据模型，设计用户表结构
- 实现用户登录接口，使用 Token 机制进行身份验证
- 配置 Spring Security 权限验证，实现基于角色的访问控制（管理员/医生）
- 实现登录拦截器，验证请求头中的 Token 有效性

**2. CodeArts 项目代码检查（1小时）**
- 使用 CodeArts 代码检查功能对已有代码进行静态分析
- 修复代码规范问题，优化代码质量

**3. 医师模型创建及查询接口实现（3小时）**
- 创建 DoctorModel、DoctorLevelModel、TreatTypeModel 等数据模型
- 创建 DoctorParam 参数类和 DoctorEntity 实体类
- 实现 DoctorMapper 接口和 XML 映射文件，编写医师查询 SQL（支持分页和关键字搜索）
- 实现 DoctorService 分页查询方法和 DoctorController 查询接口

**4. 医师添加、修改、删除及重置密码接口实现（6小时）**
- 实现医师信息的增删改接口
- 实现医师密码重置功能，默认重置为初始密码
- 在 Service 层添加业务逻辑校验
- 使用 Postman 测试各接口的正确性

**5. 医药公司模型创建及查询接口实现（3小时）**
- 创建 DrugCompany 实体类和 DrugCompanyEntity
- 实现 CompanyMapper 接口和 XML 映射文件
- 实现 CompanyService 分页查询和 CompanyController 查询接口

**6. 医药公司添加、修改、删除接口实现（6小时）**
- 实现医药公司信息的增删改接口
- 处理医药公司与医药公司政策的关联删除逻辑
- 完成接口测试

**收获：** 本周深入实践了 Spring MVC + MyBatis 的后端开发模式，掌握了 RESTful API 的设计规范，熟练使用 PageHelper 实现分页查询，理解了 Token 认证和角色权限控制的实现原理。

## 二、劳动任务和体会

本周编写了大量的后端代码，包括实体类、Mapper 接口、Service 服务层和 Controller 控制层。通过实际编码，深刻理解了 MVC 分层架构的优势——各层职责清晰，代码易于维护和扩展。同时也体会到了单元测试和接口测试的重要性，每完成一个接口都要及时测试验证。
