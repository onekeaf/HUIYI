## 1. 实习所在单位、岗位和工作内容简介

本次实习在四川青软优加科技有限公司开发部担任软件开发工程师岗位，参与医药信息管理系统的全栈开发工作。

四川青软优加科技有限公司是一家专注于软件技术服务与人才培养的科技企业，致力于为医疗健康行业提供信息化解决方案。公司开发部主要负责各类管理信息系统的设计、开发与维护工作。

本次实习的核心工作内容是参与医药信息管理系统的设计与开发。该系统是一套面向医药行业的综合管理平台，旨在实现医药公司、药品、销售地点、医生、医保政策等信息的统一管理与展示。系统采用前后端分离架构，后端基于 Spring Boot + MyBatis + MySQL 技术栈，前端基于 Vue.js + Element UI + Vuex 技术栈。

我的主要工作职责包括：参与项目需求分析和数据库设计；负责后端 RESTful API 接口的开发，涵盖用户登录认证、医生管理、医药公司管理、药品管理、销售地点管理、政策管理、必备材料管理等多个业务模块；负责前端页面的设计与开发，实现各管理模块的增删改查功能界面；完成数据面板功能，使用 ECharts 实现数据可视化展示；集成高德地图 API 实现销售地点的地图展示功能；参与项目的编译构建、部署测试和最终验收工作。

## 2. 劳动教育小结

在为期八周的实习过程中，我深入参与了医药信息管理系统的全栈开发工作，这不仅是一次技术实践，更是一次深刻的劳动教育体验。

**参与的劳动活动方面**，我的劳动贯穿了软件项目的完整生命周期。在项目准备阶段，我参与了需求分析文档、概要设计文档和数据库设计文档的编写工作，这些看似"纸上谈兵"的文档工作实际上是软件开发中不可或缺的脑力劳动。在编码实现阶段，我完成了后端十九个任务模块的接口开发和前端十个页面模块的设计实现，累计编写了大量的 Java 后端代码和 Vue.js 前端代码。在功能扩充阶段，我独立完成了数据面板的 ECharts 图表绘制和高德地图的集成开发。在部署测试阶段，我参与了项目的编译构建、沙箱部署和云端测试工作。

**劳动收获方面**，通过这次实习，我收获了扎实的全栈开发技能。后端方面，我熟练掌握了 Spring Boot 框架的使用，能够独立完成 RESTful API 的设计与实现，理解了 Token 认证、角色权限控制、分页查询等核心技术。前端方面，我熟练使用 Vue.js + Element UI 进行组件化开发，掌握了 Vuex 状态管理、Vue Router 动态路由等技术。此外，我还学习了 ECharts 数据可视化、高德地图 API 集成、Jackson JSON 解析等实用技能。更重要的是，我理解了前后端分离架构的设计思想，掌握了 Git 版本控制和团队协作开发的工作流程。

**劳动心得与体会方面**，这次实习让我深刻认识到软件开发是一项需要耐心、细心和恒心的劳动。在编写后端接口时，每一个 SQL 语句都需要反复调试验证，每一个接口都需要通过 Postman 进行测试确认。在开发前端页面时，每一个交互细节都需要精心打磨，每一个表单验证规则都需要考虑周全。我体会到了"工匠精神"在软件开发中的重要性——代码不仅要能运行，还要写得规范、易读、易维护。

同时，我也深刻体会到了团队协作的重要性。在项目开发过程中，前后端的接口对接需要充分沟通，代码的版本管理需要严格规范，阶段性的评审需要认真准备。每一个环节都离不开团队成员的密切配合。通过三次阶段评审，我学会了如何清晰地表达自己的工作成果，如何虚心接受他人的建议，如何在团队中发挥自己的价值。

这次劳动实践让我明白，软件开发不是简单的"敲代码"，而是一项融合了创造力、逻辑思维和工程素养的综合性劳动。每一行代码背后都凝聚着开发者的思考和汗水，每一个功能的实现都是团队协作的结晶。

## 3. 个人小结

### 一、实习任务完成情况

在为期八周的实习中，我按照项目计划圆满完成了所有开发任务，累计投入约 200 小时的有效工作时间。

**项目准备阶段（第1周）**，我参与了项目启动会议，完成了需求分析、概要设计和数据库设计工作。通过深入分析业务需求，设计了包含用户表、医生表、药品表、医药公司表、销售地点表、城市表、医保政策表、医药公司政策表、必备材料表、权限菜单表等在内的完整数据库结构，为后续开发奠定了基础。

**环境搭建阶段（第2周）**，我完成了 CodeArts 项目工程创建、后端 Spring Boot 项目初始化、前端 Vue.js 项目初始化、代码仓库创建和 IDE 代码托管配置等工作，并通过了第一阶段评审。

**后端编码实现阶段（第3-5周）**，这是工作量最大的阶段，我完成了 19 个后端任务模块的开发：
- 实现了用户登录认证和权限验证机制，采用 Token + 角色权限的方式控制接口访问
- 完成了医师信息管理模块，包括医师的增删改查和密码重置功能
- 完成了医药公司管理模块，包括公司信息和公司政策的增删改查功能
- 完成了城市信息管理模块，支持城市信息的添加、删除和查询
- 完成了医保政策管理模块，实现政策信息的增删改查功能
- 完成了销售地点管理模块，实现药店信息的增删改查功能
- 完成了药品信息管理模块，包括药品的增删改查和图片上传功能
- 完成了必备材料管理模块，实现材料信息的增删改查功能
- 完成了用户菜单权限管理，根据用户角色动态返回菜单列表

**前端编码实现阶段（第5-7周）**，我完成了 10 个前端页面模块的开发：
- 实现了登录页面和前端登录逻辑，包括 Token 存储和路由守卫
- 实现了动态路由渲染机制，根据后端返回的菜单列表动态注册路由
- 完成了医药公司、销售地点、城市信息、药品信息、医保政策、医药公司政策、医生信息、必备材料等 8 个管理模块的前端页面，每个模块都包含搜索、表格展示、分页、新增/修改/删除弹窗等完整功能

**功能扩充阶段（第7周）**，我完成了两个亮点功能：
- 数据面板功能：后端使用 Jackson 构建 JSON 数据，聚合医师人数、药品种类、合作企业、入驻药店等统计信息；前端使用 ECharts 绘制医生职级柱状图和医院科室饼图，展示最新政策信息
- 销售地点地图展示：后端扩展数据库表结构添加经纬度字段；前端集成高德地图 API，实现地图标记展示、点击新增和修改销售地点、逆地理编码获取地址等功能

**部署测试阶段（第8周）**，我完成了项目的编译构建、沙箱部署和云端测试工作，并通过了第三阶段评审和项目最终验收。

### 二、取得成果

通过本次实习，我独立完成了一套完整的医药信息管理系统，该系统具备以下特点：
- 采用前后端分离架构，后端提供标准化的 RESTful API，前端通过 Axios 进行接口调用
- 实现了基于 Token 的用户认证和基于角色的权限控制
- 支持 10 个业务模块的增删改查操作，覆盖医药行业的核心管理需求
- 集成 ECharts 数据可视化，直观展示业务统计数据
- 集成高德地图 API，实现销售地点的地图化管理
