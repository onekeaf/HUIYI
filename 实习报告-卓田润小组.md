## 一、项目介绍

### 1、项目目的与要求

**项目目的：**

本项目旨在设计并开发一套医药信息管理系统，为医药行业提供一个集医药公司管理、药品管理、销售地点管理、医生信息管理、政策管理等功能于一体的综合管理平台。通过本项目的开发实践，达到以下目的：

- 掌握前后端分离架构的设计思想和开发流程
- 熟练运用 Spring Boot + MyBatis 进行后端 RESTful API 开发
- 熟练运用 Vue.js + Element UI + Vuex 进行前端单页面应用开发
- 理解 Token 认证和角色权限控制的实现原理
- 学习 ECharts 数据可视化和高德地图 API 集成等扩展技能
- 掌握软件项目从需求分析到部署上线的完整开发流程

**项目要求：**

- 系统采用 B/S 架构，前后端分离开发
- 后端使用 Spring Boot 框架，提供标准化的 RESTful API
- 前端使用 Vue.js 框架，实现响应式单页面应用
- 数据库使用 MySQL 8.0，使用 MyBatis 进行数据持久化
- 实现用户登录认证和基于角色的权限控制
- 实现不少于 8 个业务管理模块的增删改查功能
- 扩展实现数据可视化和地图展示功能
- 代码规范、注释清晰、结构合理

## 二、项目内容

### 1、需求分析

#### 1.1 系统概述

医药信息管理系统是一套面向医药行业的综合管理平台，旨在实现医药公司、药品、销售地点、医生、医保政策、必备材料等信息的统一管理与展示。系统支持管理员和医生两种角色，通过权限控制实现功能差异化。

#### 1.2 系统角色分析

| 角色 | 权限描述 |
|------|---------|
| **管理员（utype=1）** | 拥有所有功能模块的增删改查权限，可管理医生账号、重置密码、管理所有业务数据 |
| **医生（utype=2）** | 拥有部分模块的查询权限，可查看必备材料、药品信息等，不可进行数据修改操作 |

#### 1.3 功能需求

**核心功能模块：**

- **用户登录与权限管理**：用户名密码登录、Token认证、动态菜单加载、角色权限控制
- **数据面板**：统计数据展示、ECharts柱状图（医生职级分布）、饼图（医院科室分布）、最新政策列表
- **医药公司管理**：公司信息的增删改查，支持分页和关键字搜索
- **销售地点管理**：药店信息的增删改查，高德地图展示和地图操作
- **城市信息管理**：城市信息的添加、删除、查询，省市区三级联动
- **药品信息管理**：药品的增删改查，图片上传，药品与销售地点多对多关联
- **医保政策管理**：医保政策的增删改查
- **医药公司政策管理**：公司政策的增删改查，关联医药公司
- **医生信息管理**：医生的增删改查，密码重置，级别和诊治类别管理
- **必备材料管理**：材料的增删改查，角色权限控制

#### 1.4 运行环境需求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS |
| JDK | 1.8 及以上 |
| 数据库 | MySQL 8.0 |
| Node.js | v14.0 及以上 |
| 浏览器 | Chrome 90+、Edge 90+ |
| 分辨率 | 1920×1080 及以上 |

#### 1.5 非功能性需求

- **性能需求**：普通查询接口响应时间不超过 500ms，支持分页查询，前端首屏加载不超过 3 秒
- **安全性需求**：密码加密存储，Token 认证，角色权限控制，CORS 跨域配置，路由守卫

### 2、概要设计

#### 2.1 设计思想

系统采用**前后端分离**的 B/S 架构设计。前端作为独立的单页面应用（SPA），通过 HTTP 请求调用后端提供的 RESTful API 获取数据；后端专注于业务逻辑处理和数据持久化，以 JSON 格式返回数据。这种架构的优势在于：

- **职责分离**：前后端独立开发、独立部署，降低耦合度
- **技术灵活**：前后端可以独立选择最适合的技术栈
- **并行开发**：前后端可以同时开发，提高开发效率
- **易于维护**：各层职责清晰，代码易于理解和维护

#### 2.2 实现方法

- **后端**：采用 Spring Boot 框架，遵循 Controller → Service → Mapper 三层架构。Controller 层负责接收请求和返回响应，Service 层负责业务逻辑处理，Mapper 层负责数据库操作。使用 MyBatis 作为 ORM 框架，通过 XML 映射文件编写 SQL 语句。使用 PageHelper 插件实现分页查询，使用 Jackson 进行 JSON 数据构建。
- **前端**：采用 Vue.js 框架，使用 Element UI 组件库构建管理后台界面。使用 Vuex 进行全局状态管理，Vue Router 进行路由管理，Axios 进行 HTTP 请求。使用 ECharts 进行数据可视化，高德地图 JS API 进行地图展示。

**动态路由渲染**：根据用户角色从后端获取菜单列表，前端动态注册路由：

```javascript
function tree(data, arr) {
  data.forEach((datas, index) => {
    arr.push({
      path: datas.path,
      name: datas.name,
      component: datas.component == "Layout"
        ? Layout
        : () => import(`../views/${datas.component}/index.vue`),
      meta: { title: datas.meta.title },
      children: [],
    });
    if (datas.children) {
      arr[index].children = tree(datas.children, []);
    }
  });
  return arr;
}
```

## 三、项目总结

### 1、项目成果

经过为期 24 天的开发实践，成功完成了医药信息管理系统的全栈开发工作。系统实现了以下核心功能：

- **10 个业务管理模块**：涵盖医药公司、销售地点、城市信息、药品信息、医保政策、医药公司政策、医生信息、必备材料等管理功能

- **高德地图集成**：实现销售地点的地图化管理，支持地图标记、点击新增和修改
- **完善的权限控制**：基于 Token 认证和角色权限的访问控制机制
- **动态路由系统**：根据用户角色动态加载菜单和路由

### 2、技术收获

通过本次项目实践，掌握了以下核心技术：

- **后端**：Spring Boot、MyBatis、PageHelper、Jackson、RESTful API 设计、Token 认证、角色权限控制
- **前端**：Vue.js、Element UI、Vuex、Vue Router、Axios、ECharts、高德地图 JS API
- **工程化**：Git 版本控制、前后端分离架构、项目编译构建与部署

### 3、经验与反思

- **前期设计的重要性**：充分的需求分析和数据库设计能够大幅减少后期的返工和修改
- **代码规范的价值**：遵循 MVC 分层架构和 RESTful 设计规范，使代码结构清晰、易于维护
- **持续学习的必要性**：ECharts 和高德地图等第三方库的学习和集成，拓展了技术视野
- **测试驱动的意识**：每完成一个接口都应及时测试验证，避免问题积累

### 4、不足与改进方向

- 系统未实现完整的单元测试覆盖，后续可引入 JUnit 和 Mockito 进行自动化测试
- 前端代码可进一步组件化，抽取更多公共组件减少重复代码
- 安全性方面可引入 Spring Security 框架进行更完善的权限管理
